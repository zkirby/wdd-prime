Question 2: WDD Prime

Relevant readings: classes (take a good look at get/set hint hint), symbols (don't read too in-depth, we will come back to symbols, just look at Symbol.toPrimitive).

PREAMBLE:
Now that you have your footing with JavaScript, we will spend the rest of the homeworks developing a hip new online grocery store to compete with Amazon Prime!

Our online store allows anyone to create a 'market' where they can add food produce they want to sell on the world wide web and also visit other people's markets. The owner of a market is called a 'farmer'

Here's why it's better than Amazon Prime: Everytime a farmer adds a food item to sell on their market, WDD Prime automatically discounts all previously added item's by 5%!

Today we will be working on the 'admin' panel for a farmer's market, but later we will add a login system, a way to view other markets, the ability to 'watch' items from other markets and automatically purchase the item when it dips below a certain price, and a few other cool features!

TASK: We will be essentially making our own custom html tag via a class. Fill in the Item class so that it satisfies the following properties:

  1. An item must have a way to get and set nutrients such that item.nutrients returns "[food name] has nutrients: [nutrient 1], [nutrient 2], ..." and item.nutrients = new_nutrients adds all the nutrients in new_nutrients to the nutrients attribute of the item. If new_nutrients is a single item (i.e. a string not a list), it simply adds that single item to list
  2. An item must have a way to get and set the price of an item such that item.price returns the price of that item and item.price = item.price updates the price to be the *new* discounted by 5% price (note that a 5% discount means new_price = old_price - old_pirce * 5%)
  3. Two items added together returns the combined price of the items. E.g. t = new Item('t', 3, ''); g = new Item('b', 4, ''); console.log(t + b) prints 7 to the console
  4. Finally, the item must have a render method. The render method must return an HTML String (i.e. a string of valid html such as "<div> Hello </div>") that has the exact following structure:

    • div w/ class 'card-item', optional class 'active_class' when an item is in the shopping cart, and data-id that is equal to the id of the item (i.e. data-id=[the item's id])
    • -> a nested div with class 'card-name' and the text = the name of the item
    • -> a nested div with class 'card-price' and the text = the price of the item
    • -> this nested div: "<div class='delete-item'>×</div>"
    • -> this nested div: "<div class='add-to-cart'></div>"
